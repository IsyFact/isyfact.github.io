Negativliste

Diese Seite ist ein Teil der IsyFact-Standards. Alle Inhalte der Seite, insbesondere Texte und Grafiken, sind urheberrechtlich geschützt. Alle urheberrechtlichen Nutzungs- und Verwertungsrechte liegen beim Bundesverwaltungsamt.

Die Nutzung ist unter den Lizenzbedingungen der Creative Commons Namensnennung 4.0 International gestattet.

1. Standards

Kategorie / Anwendungsbereich Hersteller / Produkt Version Seit Release Status Begründung der Entscheidung

Austauschformate für Bilder

BMP

1.0.0

verworfen

Dieses kompressionslose Bildformat sollte wegen der entstehenden großen Bilddateien nicht verwendet werden. Siehe SAGA 4.0 Blacklist

Austauschformate für Bilder

PCX

1.0.0

verworfen

Dieses Bildformat ist im World Wide Web unüblich und wird nicht von allen Browsern unterstützt. Siehe SAGA 4.0 Blacklist

Austauschformate für Audio- und Videodateien

MPEG-2

1.0.0

verworfen

Dieses Format benötigt für eine ansprechende Qualität Bandbreiten von 3-8 MBit/s. Für die Zielgruppen von E-Government-Anwendungen muss davon ausgegangen werden, dass nur wenige Anwender über einen solch breitbandigen Internetzugang verfügen, zum Vergleich: ISDN hat eine Bandbreite von 64 kBit/s. Siehe SAGA 4.0 Blacklist

Austauschformate für Audio- und Videodateien

DivX v6

1.0.0

verworfen

Zu DivX v6 existiert keine frei verfügbare Dokumentation des Codecs sowie eine (möglichst) freie Nutzbarkeit. Weiterhin steht DivX v6 nur für Windows und MacOS zur Verfügung. Siehe SAGA 4.0 Blacklist

Austauschformate für Geoinformationen

VML

1.0.0

verworfen

VML ist eine seit 1998 nicht mehr weiterentwickelte Technologie zum Anzeigen von Geoinformationen und sollte nicht zum Datenaustausch verwendet werden. (Vector Markup Language). Siehe SAGA 4.0 Blacklist

Middleware-Kommunikation mit verwaltungsexternen Applikationen

RMI

1.0.0

verworfen

Für die Kommunikation vom Server zum Client, also zum Rechner des Anwenders von Online-Dienstleistungen, ist RMI-Kommunikation ungeeignet, da Umkonfigurationen von Firewalls der Netzwerke des Client notwendig werden können. Für die Kommunikation von Server zu Server innerhalb der Verwaltung ist RMI ein SAGA-Standard, siehe SAGA 2.0, Abschnitt 8.6.1.1, Seite 93. Siehe SAGA 4.0 Blacklist

Middleware-Kommunikation mit verwaltungsexternen Applikationen

RMI-IIOP

1.0.0

verworfen

Für die Kommunikation vom Server zum Client, also zum Rechner des Anwenders von Online-Dienstleistungen, ist RM-IIOP-Kommunikation ungeeignet, da Umkonfigurationen von Firewalls der Netzwerke des Clients notwendig werden können. Für die Kommunikation von Server zu Server innerhalb der Verwaltung ist RMI-IIOP ein SAGA-Standard, siehe SAGA 2.0, Abschnitt 8.6.1.1., Seite 93. Siehe SAGA 4.0 Blacklist

3. Bibliotheken für die Anwendungsentwicklung

Kategorie / Anwendungsbereich Hersteller / Produkt Version Seit Release Status Begründung der Entscheidung

Bildbearbeitung

JAI

1.1.3

1.0.0

verworfen

Bei den Lasttests ist ein Speicherleck in der Konvertierungsbibliothek "Java Advanced Imaging Image I/O" aufgefallen. Ein Bugfix existiert bereits auf den Webseiten des JAI-Projekts, allerdings gibt es noch keine neue Stable-Version der Bibliothek. Wir haben daher einen aktuellen Daily-Build der Software verwendet (Version vom 26.08.2009). Der funktionale Test und der Lasttest haben keine Fehler bei den von uns verwendeten Konvertierungsfunktionen aufgezeigt. Es ist vom Einsatz des Release 1.1.3 abzusehen

JSF-Dialogsteuerung

Spring Webflow

< 2.0.6

1.0.0

verworfen

  • Probleme bei Last und Konkurrierenden Zugriffen (Threading-Problem).

  • Zahlreiche Versionen (mindestens 2.0.5, 2.0.6) enthalten einen Bug in Bezug auf das Post-Redirect-Get-Pattern (siehe https://jira.springsource.org/browse/SWF-1096).

Collection-API

Commons-Collection

verworfen

  • Weiterentwicklung fraglich. Es muss Guava eingesetzt werden.

  • Hinweis: Commons Collections besitzt in folgenden Versionen zudem einen sicherheitskritischen Fehler (vgl. https://issues.apache.org/jira/browse/COLLECTIONS-580):

    • >=3.0 und <3.2.2

    • >=4.0 und <4.1

JSF-Dialogsteuerung

Spring Webflow

2.4.4

1.3.6

verworfen

Enthält einen Fehler, der in Abhängigkeit zur Größe des JSF-Komponentenbaums zu stark erhöhten Laufzeiten bei POST-Requests führt.

JSF-Dialogsteuerung

Spring Webflow

<2.4.8

1.6.0

verworfen

Enthält einen Fehler in der Session-Persistierung und verhindert den Einsatz von Partial State Saving in JSF-GUIs.

Excel-Datenzugriff

JExcelAPI

2.6.x

1.0.0

verworfen

Wird nicht weiterentwickelt und soll nicht mehr verwendet werden. Für neue Entwicklungen sollte Apache POI verwendet werden, dass, aktiv weiterentwickelt wird und z.B. auch Office 2007-Formate unterstützt.

Bean Mapping

Dozer

5.3.2

1.0.0

verworfen

Kritischer Fehler https://github.com/DozerMapper/dozer/issues/118 führt dazu dass Dozer komplett ausfällt.

Kryptographie

BouncyCastle

⇐ 1.5

1.2.2

verworfen

Kritischer Fehler in Version ⇐ 1.5 http://web.nvd.nist.gov/view/vuln/detail?vulnId=CVE-2015-7940 ermöglicht Man-In-The-Middle-Angriffe.

JSF-Implementierung

Oracle Mojarra JavaServer Faces

<2.2.13

1.3.6

verworfen

Einsatz führt zu irritierenden Warnings im wrapper.log (vgl. https://java.net/jira/browse/JAVASERVERFACES-3746).

HTTP-Client

Netty

2.4.0

verworfen

Der Eintrag gilt ausschließlich für die Verwendung von Netty als HTTP-Client in Verbindung mit Spring WebFlux. Hier sind aufgrund früherer Architekturentscheidungen ausschließlich Produkte zu verwenden, die kompatibel zum Produkt Apache Tomcat sind und keinen zweiten Server neben dem Tomcat aufbauen.
